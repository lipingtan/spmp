## 3. 功能需求

### 3.1 基础数据管理

管理物业的地理层级数据，为所有业务模块提供基础支撑。

功能清单：
- 片区管理：新增/编辑片区，将小区分配到片区
- 小区管理：新增/编辑/停用小区，关联片区
- 楼栋管理：在小区下新增/编辑楼栋
- 单元管理：在楼栋下新增/编辑/删除单元
- 房屋管理：新增/编辑房屋，支持 Excel 批量导入，按多级条件筛选
- 级联查询：提供小区→楼栋→单元→房屋的级联选择数据

数据层级：片区 → 小区 → 楼栋 → 单元 → 房屋

### 3.2 用户与权限管理

管理系统用户账号和权限体系，支持 RBAC 模型。

功能清单：
- 登录认证：用户名密码登录，JWT Token 认证，密码错误锁定
- 用户管理：新增/编辑/禁用用户，分配角色，重置密码
- 角色管理：新增/编辑角色，分配菜单权限，配置数据权限级别
- 菜单管理：三级菜单结构（目录→菜单→按钮），权限标识管理
- 数据权限：四级控制（全部→片区→小区→楼栋→仅本人），通过 SQL 拦截器自动过滤
- 个人中心：查看个人信息，修改密码

预置角色：超级管理员、物业管理员、片区经理、楼栋管家、维修人员、业主

### 3.3 业主管理

管理业主信息和房产绑定关系。

功能清单：
- 业主信息：录入/编辑业主（姓名、手机号、身份证号），支持搜索
- 房产绑定：将业主绑定到房屋，支持业主/租户/家属三种关系类型
- 家庭成员：业主可添加家庭成员，自动关联房产，每套房产最多 10 人
- 业主认证：业主通过 H5 端注册→选择房屋→提交认证申请→管理端审批
- 未认证限制：未认证业主只能查看公告，不能报修和缴费

### 3.4 工单管理

报修工单全生命周期管理。

功能清单：
- 业主报修：选择报修类型（水电/门窗/管道/公共设施/其他），填写描述，上传图片（最多5张）
- 工单派发：楼栋管家/物业管理员将工单分配给维修人员
- 维修处理：维修人员接单、上传维修照片、填写处理说明、完成维修
- 业主验收：验收通过/不通过，满意度评分（1-5星）
- 自动完成：超过 7 天未验收自动完成
- 工单统计：待处理数、处理中数、完成数、平均处理时长、满意度

工单状态流转：待派发 → 待接单 → 处理中 → 待验收 → 已完成（任意阶段可取消）

### 3.5 缴费管理

物业费用的账单和支付管理。

功能清单：
- 费用配置：按小区配置收费标准（物业费单价等）
- 账单生成：按小区批量生成月度物业费账单（金额=面积×单价），支持手动创建单条账单
- 在线缴费：业主查看待缴账单，支持单条/多条合并支付（微信/支付宝）
- 缴费催收：逾期账单列表，批量发送催收通知
- 费用减免：对特殊账单进行部分或全额减免
- 收费统计：按月/季/年统计应收、实收、收缴率，按小区/楼栋/费用类型维度

账单状态：待缴费 → 已缴费 / 已逾期 / 已减免

### 3.6 公告管理

社区公告发布和推送。

功能清单：
- 发布公告：填写标题、内容（富文本）、类型（通知/紧急/活动）、推送范围（全部/指定小区/指定楼栋）
- 置顶和有效期：支持设置置顶、有效期
- 业主查看：业主只能看到自己所在小区/楼栋的公告，自动标记已读
- 未读提示：未读公告数量角标
- 已读统计：查看每条公告的已读/未读人数和比例，支持对未读业主重新推送
- 撤回公告：支持撤回已发布的公告

### 3.7 门禁管理

访客通行管理。

功能清单：
- 访客预约：业主为来访客人预约门禁通行，生成通行二维码（有效期：预约时间前后各2小时）
- 临时通行证：楼栋管家/物业管理员为临时来访人员发放通行证（1小时/4小时/当天）
- 访客审批（可选）：按小区配置是否需要楼栋管家审批
- 通行记录：查询门禁通行记录，支持按时间、小区、访客筛选，支持导出
- 二维码验证：门禁设备扫码验证通行权限

## 4. 非功能性需求

### 4.1 性能

- 核心接口响应时间 P99 < 500ms
- 级联查询接口 < 200ms（使用 Redis 缓存）
- 二维码验证接口 < 100ms
- 支持并发用户数 ≥ 300

### 4.2 安全

- 密码 BCrypt 加密存储
- JWT Token 认证（access_token 2h + refresh_token 7d）
- 登录限流：同一 IP 每分钟最多 10 次，密码错误 5 次锁定 30 分钟
- 身份证号加密存储、脱敏展示
- 支付接口幂等、回调验签
- 公告富文本 XSS 过滤
- 二维码加密防伪造

### 4.3 兼容性

- PC 端：Chrome 90+、Edge 90+
- H5 端：微信内置浏览器、主流移动浏览器

### 4.4 可用性

- 系统可用性 ≥ 99.9%

## 5. 数据权限模型

系统采用四级数据权限控制，不同角色看到不同范围的数据：

| 权限级别 | 数据范围 | 适用角色 |
|----------|----------|----------|
| 全部 | 所有数据，不做过滤 | 超级管理员 |
| 片区 | 所辖片区下所有小区的数据 | 片区经理 |
| 小区 | 所管辖小区的数据 | 物业管理员 |
| 楼栋 | 所负责楼栋的数据 | 楼栋管家 |
| 仅本人 | 只能看自己相关的数据 | 维修人员（自己的工单）、业主（自己的房产） |

数据权限通过 MyBatis 拦截器自动注入 SQL 过滤条件，业务代码无需手动处理。

## 6. 首次初始化流程

系统未初始化时，后端仅开放初始化相关接口（`/api/v1/init/*`），核心接口包括：

- `GET /api/v1/init/status`
- `POST /api/v1/init/test-connection`
- `POST /api/v1/init/execute`

### 6.1 操作步骤

1. 启动后端与前端；
2. 打开前端页面：`http://localhost:3000`（进入初始化页面 `InitPage`）；
3. 填写 MySQL 与 Redis 连接信息；
4. 点击“测试连接”；
5. 点击“执行初始化”；
6. 初始化完成后**手动重启后端服务**。

初始化接口访问地址示例（后端）：`http://localhost:8080/api/v1/init/status`

### 6.2 初始化产物

- `config/init-config.yml`：初始化后的连接配置（密码加密）
- `config/init.lock`：初始化完成标记

## 8. 常见问题

### 8.1 初始化成功后仍无法正常访问业务接口

请确认是否已重启后端。初始化完成后必须重启，系统才会加载新配置并退出最小启动模式。

### 8.2 前端请求接口失败

请确认：

- 前端是否运行在 `3000`
- 后端是否运行在 `8080`
- API 前缀是否为 `/api/v1`
- 访问地址是否正确（前端 `http://localhost:3000`，后端 `http://localhost:8080/api/v1`）

### 8.3 登录失败

请先确认使用默认管理员账号（`admin / Spmp@2026`），并排除数据库中密码已变更情况。
